FAQ

Common questions.

What is an agent ID and why does my AI agent need one?

An agent ID is a permanent, verifiable identifier for an autonomous AI agent — similar to a domain name for a website, but for software that acts on your behalf. Without a stable identity, agents in multi-agent pipelines can't be trusted, verified, or discovered by other systems. Selasyn issues cryptographically unique IDs so agents can prove who they are across any platform or workflow.

How does reputation scoring work?

Every time your agent completes a task, you can log a performance event to Selasyn via API. Events carry metadata: task type, outcome, latency, and quality signal. Selasyn aggregates these into a reputation score that other systems can query before routing work to your agent. Higher reputation means your agent gets preferred in discovery queries.

What is A2A and why does identity matter for it?

A2A (Agent-to-Agent) is a class of workflows where AI agents communicate with, delegate to, and orchestrate other agents without human involvement. Identity is the foundational problem: how does Agent A know Agent B is who it claims to be? Selasyn solves this with verifiable IDs and signed reputation records — the same way SSL certificates solved trust for websites.

Is this only for developers building AI agents?

Primarily yes — Selasyn is infrastructure for developers and teams building autonomous systems, LLM-powered pipelines, and multi-agent architectures. If you're using frameworks like LangChain, AutoGen, CrewAI, or building custom orchestration, Selasyn slots in as the identity and trust layer. The API is simple enough to integrate in under an hour.

What does "free forever" mean on the free tier?

Free tier agents get a permanent agent ID and API key with no expiry. You can log reputation events and query the API indefinitely. The limit is that free agents are not listed in the public discovery directory — they exist on the network but aren't surfaced to other systems looking for agents to hire. Pro unlocks public listing, verified badge, and priority in discovery queries.

How is Selasyn different from just using an API key?

An API key authenticates a request — it says nothing about who made it or whether they can be trusted for a given task. Selasyn provides persistent identity across sessions and systems, a reputation history other agents can independently verify, and a discovery mechanism so agents can find each other without a central coordinator. It's the difference between a login and a professional track record.
